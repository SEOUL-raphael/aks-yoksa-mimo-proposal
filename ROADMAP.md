# Roadmap

## 1단계: 원문 공개

- legalize 스타일의 `kr/{서명}/{권책}/{기사}` 트리 제공
- 서지, 해제, 권책, 기사 원문 보존
- `_index/books.jsonl`, `_index/articles.jsonl` 제공

## 2단계: 번역 1차 적재

- SQLite 작업 큐 기반으로 전체 기사에 대해 번역 시도
- 성공은 `ok`, 실패는 `failed`로 기록
- 실패 사유를 보존해 모델별 재처리 근거로 활용

## 3단계: MiMo 추가 활용

- 실패 유형별 프롬프트 분기
- 난독 문자/이체자 포함 문장 우선 재처리
- 독음, 자연어 번역, 의미 요약, 엔티티 후보를 JSON으로 생성

## 4단계: 색인과 지식 그래프

- 한문 원문 FTS
- 한국어 번역 FTS
- 인명, 지명, 관직명, 문헌명 엔티티 색인
- 원문 증거 기반 노드/엣지 생성

## 5단계: 검수와 공개 버전 관리

- 번역 실패 재처리
- 한글/한자 외 문자 품질 점검
- 검수 완료 번역만 별도 릴리스 태그로 공개
- GitHub Pages 또는 API 형태의 탐색 UI 제공
