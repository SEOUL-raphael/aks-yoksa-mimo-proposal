# AKS Yoksa Classical Text Translation Project

## 프로젝트 개요

이 저장소는 한국학중앙연구원 장서각 `고도서 > 본문텍스트 > 서명별` 공개 목록을 기준으로, 고문헌 원문을 `legalize-kr`와 유사한 경로 기반 트리 구조로 정돈한 진행형 데이터셋이다.

현재 1차 목표는 한문 원문을 canonical 데이터로 보존하면서, 한국어 번역, 독음, 의미 요약, 인명/지명/관직명/문헌명 엔티티를 파생 레이어로 누적하는 것이다. 원문은 검증과 재처리를 위해 절대 덮어쓰지 않고, 번역 결과는 성공과 실패를 모두 상태값으로 남긴다.

## 현재 범위

- 원천: `http://yoksa.aks.ac.kr/jsp/aa/BookList.jsp?fcs=s&fcsd=st`
- 서지: 439건
- 권책: 1,325건
- 기사: 113,150건
- 본문 있는 번역 대상: 112,287건
- 저장 구조: `kr/{서명}/{권책}/{기사}.md` 및 대응 JSON

## 데이터 원칙

- 한문 원문을 canonical로 유지한다.
- 한국어 번역, 독음, 의미 요약은 파생 데이터로 저장한다.
- 실패 결과도 `status=failed`와 오류 사유를 남겨 재처리 근거로 사용한다.
- 장기적으로 한문 원문 FTS, 한국어 번역 FTS, 엔티티 색인, 지식 그래프를 분리해 하이브리드 검색을 구성한다.

## MiMo 활용 필요성

전체 대상이 11만 건 이상이고, 각 문헌은 이체자, 난독 문자, 관직명, 지명, 의례 용어가 섞인 한문 자료다. 일반 번역 모델만으로는 JSON 형식 준수, 한글 독음 생성, 의미 요약, 엔티티 추출의 품질이 불안정하다.

MiMo 토큰은 다음 작업에 사용하려고 한다.

- 실패 번역의 원인별 재처리
- 한문 원문에 대한 한국어 독음 생성
- 고문헌 문맥에 맞는 자연스러운 한국어 번역
- 인명, 지명, 관직명, 문헌명 후보 추출
- 불확실 용어 표시와 후속 검수 큐 생성
- 지식 그래프용 노드/엣지 후보 생성

## 현재 파이프라인

1. 원문 수집: 서지, 해제, 권책, 기사 본문을 보존한다.
2. 트리 정돈: 사람이 읽을 수 있는 Markdown과 기계 처리용 JSON을 함께 둔다.
3. 번역 큐: SQLite 작업 큐에서 `pending/running/ok/failed` 상태를 관리한다.
4. 번역 실행: 병렬 worker가 번역을 시도하고 결과를 SQLite 및 JSONL에 적재한다.
5. 색인: SQLite FTS와 지식 그래프 테이블을 재생성한다.

## 공개 목적

이 저장소는 완성본이 아니라 진행 중인 공개 작업물이다. MiMo 신청 시점에는 원문 트리와 현재 번역 진행 상태를 근거로 제시하고, 토큰 확보 후 번역 및 품질 개선 결과를 같은 구조에 계속 반영한다.
